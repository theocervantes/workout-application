18. ACCESSIBILITY NOTES

Considerations for inclusive design and usability.

  • High contrast ratios maintained throughout (WCAG AA compliant)
  • Focus states visible on all interactive elements
  • Semantic HTML structure for screen readers
  • Touch targets minimum 44x44px for mobile
  • Color not used as sole indicator of state
  • Font sizes scalable and readable at all viewport sizes

19. USAGE GUIDELINES

Best practices for implementing the design system.

Component Hierarchy
  • Use Primary buttons for main actions (Start Workout, Save, Submit)
  • Use Secondary buttons for supporting actions (View Details, Edit)
  • Use Ghost buttons for tertiary actions (Cancel, Back)
  • Limit to one Primary button per screen section
Block Color Semantics
  • Blue (WU) = Warm-up / Preparation phase
  • Orange (Work) = Main workout blocks / Training sets
  • Green (FIN) = Finisher / Cool-down / Completion
Responsive Behavior
  • Card grids collapse to single column on mobile
  • Maintain 1.5rem padding on small screens
  • Stack buttons vertically on narrow viewports
  • Reduce font sizes proportionally below 640px
